# Owner Home — «Моя страница» владельца Lec7

## Общая концепция

Личная рабочая страница владельца платформы Lec7. Центр управления и принятия решений, где владелец видит текущее состояние проектов, взаимодействует с Owner Agent и фиксирует опыт работы.

**Принципы:**
- Информативность без перегрузки
- Быстрый доступ к ключевым действиям
- Фокус на принятии решений
- Связь между текущим состоянием и накопленным опытом

---

## Структура страницы (сверху вниз)

### 1. Заголовок и навигация

**Цель:** Быстрый доступ к основным разделам и понимание текущего контекста.

**Что видит владелец:**
- Название страницы: "Моя страница" или "Owner Dashboard"
- Краткая информация о пользователе (email, роль)
- Навигация: ссылки на основные разделы (Admin, Businesses, Settings)
- Кнопка выхода

**Действия:**
- Переход в другие разделы
- Выход из системы

---

### 2. Краткий статус (Status Overview)

**Цель:** Мгновенное понимание текущего состояния платформы и приоритетов.

**Что видит владелец:**
- **Активные проекты:** количество бизнесов в работе
- **Текущие задачи:** список из 3-5 наиболее важных задач (из Owner Agent или вручную)
- **Критические уведомления:** если есть проблемы, требующие внимания
- **Последние обновления:** краткая лента последних изменений (миграции, деплои, изменения в бизнесах)

**Действия:**
- Переход к детальному просмотру задач
- Отметка задач как выполненных
- Быстрый доступ к критическим уведомлениям

**Визуализация:**
- Компактные карточки с ключевыми метриками
- Цветовая индикация (зелёный — всё ок, жёлтый — внимание, красный — проблема)

---

### 3. Мои бизнесы / Проекты

**Цель:** Обзор всех бизнесов на платформе с возможностью быстрого доступа к каждому.

**Что видит владелец:**
- **Список бизнесов:** карточки с основными данными каждого бизнеса
  - Название и slug
  - Город и категория
  - Статус (активен / на паузе)
  - Количество запросов за последний месяц
  - Последняя активность
- **Фильтры:** по городу, категории, статусу
- **Поиск:** быстрый поиск по названию

**Действия:**
- Переход на страницу конкретного бизнеса
- Создание нового бизнеса
- Редактирование существующего бизнеса
- Просмотр публичной страницы бизнеса
- Фильтрация и сортировка списка

**Визуализация:**
- Сетка карточек (2-3 колонки на десктопе)
- Компактный список на мобильных
- Иконки статуса и активности

---

### 4. Owner Agent — Центр принятия решений

**Цель:** Интерактивный помощник для принятия архитектурных и продуктовых решений.

**Что видит владелец:**
- **Чат-интерфейс Owner Agent:**
  - История диалогов (список слева)
  - Текущий диалог (центр)
  - Поле ввода и кнопка отправки
- **Быстрые действия:**
  - Кнопка "Новый диалог"
  - Кнопка "Сохранить в Playbook" (после получения ответа)
  - Индикатор режима работы Agent (NEXT_STEP / CURSOR_TASK / RISK_CHECK)
- **Контекст:** краткая информация о том, какие данные из Playbook используются в текущем диалоге

**Действия:**
- Задать вопрос Owner Agent
- Создать новый диалог
- Сохранить ответ/решение в Playbook
- Просмотреть историю диалогов
- Скопировать ответ для использования в Cursor или других инструментах

**Визуализация:**
- Двухколоночный layout (диалоги слева, чат справа)
- Сворачиваемая панель диалогов на мобильных
- Бейджи режимов работы Agent
- Подсветка важных ответов

---

### 5. Последние действия / Опыт (Playbook)

**Цель:** Просмотр и управление накопленным опытом работы с платформой.

**Что видит владелец:**
- **Последние карточки опыта:**
  - Последние 10-15 записей из Playbook
  - Для каждой карточки:
    - Дата создания
    - Заголовок (title)
    - Действие (move)
    - Результат (outcome)
    - Уровень уверенности (confidence)
    - Теги
    - Scope (PLATFORM / BUSINESS)
- **Фильтры:**
  - По scope (PLATFORM / BUSINESS)
  - По confidence (LOW / MEDIUM / HIGH)
  - По тегам
  - По дате
- **Статистика:**
  - Всего карточек в Playbook
  - Распределение по scope
  - Распределение по confidence

**Действия:**
- Просмотр детальной информации о карточке
- Редактирование карточки (если нужно уточнить outcome или теги)
- Удаление карточки (если ошибочная)
- Создание новой карточки вручную
- Поиск по содержимому карточек
- Экспорт карточек (для анализа или бэкапа)

**Визуализация:**
- Список карточек с компактным отображением
- Цветовая индикация confidence (LOW — жёлтый, MEDIUM — синий, HIGH — зелёный)
- Иконки scope (PLATFORM — глобус, BUSINESS — здание)
- Возможность развернуть карточку для просмотра полной информации

---

### 6. Быстрые действия (Quick Actions)

**Цель:** Доступ к часто используемым операциям без перехода в другие разделы.

**Что видит владелец:**
- Панель с кнопками быстрых действий:
  - "Создать бизнес"
  - "Новый диалог с Agent"
  - "Добавить в Playbook"
  - "Просмотреть миграции"
  - "Проверить статус сервера"

**Действия:**
- Одно нажатие для выполнения частых операций
- Возможность настройки списка быстрых действий

**Визуализация:**
- Горизонтальная панель с иконками
- Компактное расположение в верхней части страницы

---

### 7. Футер и дополнительная информация

**Цель:** Справочная информация и ссылки на документацию.

**Что видит владелец:**
- Версия платформы
- Ссылки на документацию
- Контакты поддержки (если нужно)
- Статус последнего деплоя

**Действия:**
- Переход к документации
- Просмотр changelog

## Принципы взаимодействия

### Приоритет информации
1. **Критическое** (требует немедленного внимания) — вверху, с выделением
2. **Важное** (текущие задачи) — в центре
3. **Справочное** (история, статистика) — внизу

### Адаптивность
- **Десктоп:** все блоки видны одновременно, двухколоночный layout для Agent
- **Планшет:** компактное отображение, сворачиваемые блоки
- **Мобильный:** вертикальная компоновка, приоритет на Agent и статус

### Производительность
- Ленивая загрузка для больших списков (бизнесы, playbook)
- Кэширование данных статуса
- Оптимистичные обновления UI при действиях

---

## Связи между блоками

1. **Status Overview → Businesses:** клик по метрике ведёт к списку бизнесов
2. **Businesses → Owner Agent:** можно задать вопрос Agent о конкретном бизнесе
3. **Owner Agent → Playbook:** ответы Agent можно сохранить в Playbook
4. **Playbook → Owner Agent:** контекст из Playbook используется Agent при ответах
5. **Quick Actions → все блоки:** быстрый доступ к любому разделу
