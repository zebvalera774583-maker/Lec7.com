# Lec7 UI Kit v0.1

## 0. Принцип продукта (НЕИЗМЕНЯЕМЫЙ)

Lec7 — AI-first платформа для ведения бизнеса.

AI является главным актором интерфейса.
AI ведёт процессы.
Владелец принимает решения.

Lec7 — не маркетплейс.
Lec7 — не социальная сеть.
Lec7 — не набор инструментов с AI.

Каноническая формула Lec7:
**AI ведёт бизнес. Вы контролируете решения.**

Запрещено:
- AI как помощник
- AI как ассистент
- AI как чат-бот
- маркетинговые и хайповые формулировки

---

## 1. Цветовые роли (СМЫСЛ, НЕ КОД)

### BG — Фоновое пространство
Описание:
Светло-бежевый, почти белый, тёплый нейтральный.

Назначение:
- фон всех страниц (P1–P7)
- создаёт ощущение спокойствия и доверия
- визуально "воздух"

Запрещено:
- использовать для карточек
- использовать для активных элементов

---

### SURFACE — Рабочая поверхность
Описание:
Чисто белый.

Назначение:
- карточки
- панели
- формы
- модальные окна
- основные рабочие зоны

SURFACE всегда контрастирует с BG.

---

### SHADOW — Системная тень Lec7
Описание:
Очень лёгкая, холодная, светло-голубовато-серая тень.

Назначение:
- подчёркивает, что объект управляется системой
- визуальный маркер AI-инфраструктуры
- используется одинаково на всех карточках

Запрещено:
- чёрные тени
- разные типы теней
- декоративные тени

---

### TEXT_PRIMARY — Основной текст
Описание:
Тёмно-графитовый (не чисто чёрный).

Назначение:
- заголовки
- основной текст
- AI-действия
- CTA

---

### TEXT_SECONDARY — Вторичный текст
Описание:
Светло-серый.

Назначение:
- пояснения
- подписи
- вспомогательная информация

Запрещено:
- использовать для CTA
- использовать для AI-действий

---

### DIVIDER — Разделители
Описание:
Очень светлый серый, почти незаметный.

Назначение:
- структурирование
- ритм
- порядок

---

### AI_ACCENT — Акцент AI (КЛЮЧЕВОЙ)
Описание:
Холодный серо-синий / графитово-синий.

Назначение:
- слово "AI"
- бейджи AI
- индикаторы состояний AI
- ключевые AI-блоки

Запрещено:
- использовать как фон всей карточки
- использовать массово
- смешивать с сигнальными цветами

---

### STATE COLORS — Состояния
SUCCESS:
Спокойный зелёный — завершено

WARNING:
Тёплый янтарный — ожидание / внимание

DANGER:
Спокойный красный — ошибка / блокировка

Правило:
Состояния не смешиваются с AI_ACCENT.

---

## 2. AI-состояния (КАНОНИЧЕСКИЕ)

Разрешённые состояния (используются везде):

- AI_ACTIVE — AI ведёт процесс
- AI_COMPLETED — Выполнено AI
- AI_WAITING_DECISION — Ожидает решения владельца
- AI_ATTENTION_REQUIRED — Требует внимания
- AI_PAUSED — Приостановлено владельцем

Другие формулировки запрещены.

## 4. Базовые компоненты Lec7

Компоненты используются строго по назначению.

1. AI Hero  
   Заголовок экрана:
   "AI ведёт ваш бизнес"
   Подзаголовок:
   "Вы контролируете решения"

2. AI Badge  
   Отображение состояния AI.

3. Metric  
   Число + подпись (активные процессы, ожидания, ошибки).

4. Timeline Item  
   Лента действий AI.

5. Decision Card  
   Карточка решения:
   AI предлагает действие → владелец утверждает или изменяет.

6. Mode Switch  
   Режимы:
   - AI ведёт сам
   - AI предлагает решения
   - Ручной режим

7. Card  
   Базовая карточка (SURFACE + SHADOW).

8. Primary Button  
   Главное действие.

9. Secondary Button  
   Вторичное действие.

10. Nav Item  
    Пункт меню.

11. Section Header  
    Заголовок секции.

12. Divider  
    Разделитель.

13. Input  
    Поля ввода.

14. Empty State  
    Пустые состояния с AI-текстом.

15. Drawer / Panel  
    Выезжающие панели и меню.

## 6. Главное правило использования UI kit

Этот файл является источником истины.

При разработке:
- не менять формулировки
- не добавлять новые состояния
- не вводить новые цвета
- не "улучшать" AI без согласования

Любое отклонение от UI kit считается нарушением архитектуры продукта Lec7.
