# Обращение в поддержку Timeweb

## Тема: Проблема с Docker Hub rate limit при деплое проекта

Здравствуйте!

У меня возникла проблема при деплое проекта на VPS сервере в Timeweb.

**Описание проблемы:**

При попытке загрузить Docker образ PostgreSQL (`postgres:16` или `postgres:15`) через `docker pull` или `docker-compose pull` получаю ошибку:

```
Error response from daemon: error from registry: You have reached your unauthenticated pull rate limit. https://www.docker.com/increase-rate-limit
```

Это блокирует деплой проекта, так как без образа PostgreSQL база данных не запускается.

**Детали:**
- Сервер: Brave Ganymede (IP: 194.87.104.179)
- ОС: Ubuntu 24.04.3 LTS
- Docker версия: 29.1.3
- Docker Compose версия: v5.0.1
- Проблемный образ: `postgres:16` / `postgres:15`

**Что я уже пробовал:**
1. Замену версии образа (postgres:16 → postgres:15)
2. Ожидание 20-30 минут между попытками
3. Использование флага `--platform linux/amd64`

**Запрос:**

Можете ли вы помочь решить эту проблему? Возможные варианты:

1. **Настроить прокси или mirror для Docker Hub** на вашей стороне, чтобы обойти лимиты
2. **Предоставить доступ к альтернативному registry** для базовых образов (PostgreSQL, Node.js и т.д.)
3. **Установить образ PostgreSQL вручную** на сервер через вашу панель или поддержку
4. **Предложить альтернативное решение** для работы с Docker образами

Или, если у вас есть готовые образы PostgreSQL в вашем registry, подскажите, как их использовать.

**Дополнительная информация:**

Проект использует:
- Next.js 14 (App Router)
- PostgreSQL 16 (или 15)
- Docker Compose для оркестрации

Буду благодарен за помощь в решении этой проблемы.
